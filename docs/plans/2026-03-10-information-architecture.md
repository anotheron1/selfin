# Информационная архитектура: цели страниц

**Дата решения:** 2026-03-10

## Проблема

Dashboard стал перегруженным — содержит и оперативную информацию ("сейчас"), и детальную аналитику ("за месяц"). Блок кассового разрыва дублируется на Dashboard и Budget. Таблица план-факт в Dashboard дублирует данные, которые пользователь видит через список событий в Budget.

---

## Разделение ответственности

### Dashboard — "Прямо сейчас"

**Цель:** дать пользователю мгновенный ответ на вопрос "как дела сегодня?"

Блоки (оставить):
- Текущий баланс + прогноз конца месяца
- События сегодня + прогноз конца дня (если есть неисполненные)
- Алерт кассового разрыва (ближайший дефицит)
- Кассовый календарь — следующие дни (горизонтальный скролл)
- Прогресс-бары план/факт по категориям расходов (компактный виджет, только текущий месяц)

Блоки убрать → перенести в Analytics:
- Полная таблица отчёта план-факт (слишком детально для дашборда)
- Секция "Обязательные траты" / burn rate (детально)
- Секция "Доходы" / income gap (детально)

**Принцип:** Dashboard не должен требовать скролла в нормальной ситуации. Если событий сегодня нет — прогресс-бары и кассовый календарь.

---

### Budget — "Планирование месяца"

**Цель:** просмотр и редактирование отдельных финансовых событий; планирование по месяцам.

**Уникальная ценность:** единственная страница, где видны индивидуальные события и можно их редактировать (клик → EditEventSheet).

Блоки (оставить / добавить):
- Навигация по месяцам (любой месяц)
- Компактная шапка: итого план/факт доходов и расходов за месяц (добавить)
- Список событий сгруппированных по неделям с редактированием

Блоки убрать:
- Алерт кассового разрыва (это зона Dashboard, в Budget не нужен)

**Принцип:** Budget — это планировщик/ежедневник. Оперативные алерты здесь лишние.

---

### Analytics — "Аналитика"

**Цель:** ретроспективный анализ за несколько месяцев; детальные отчёты.

Блоки:
- Мультимесячная таблица (план/факт по категориям за 3/6/12 месяцев) — PR3
- Отчёт план-факт текущего месяца по категориям (перенесено из Dashboard)
- Burn rate обязательных трат по неделям (перенесено из Dashboard)
- Дефицит доходов (перенесено из Dashboard)

Переключатель периода: "Текущий месяц" / "3 мес" / "6 мес" / "12 мес"

При выборе "Текущий месяц" показывает детальные блоки (план-факт, burn rate, income gap).
При выборе "3 мес"+ показывает мультимесячную таблицу.

---

### Funds — "Накопления"

Без изменений. Управление копилками (целевыми фондами).

---

### Settings — "Настройки"

Без изменений. Категории, снимки бюджета.

---

## Итоговая навигация (BottomNav)

```
Дашборд | Бюджет | Фонды | Аналитика | Настройки
```

5 пунктов — нормально ложится на мобильный экран.

---

## Приоритет реализации

1. PR3 (Analytics page) — реализовать с расширенным scope: мультимесячная таблица + перенос детальных блоков из Dashboard
2. После PR3: почистить Dashboard (убрать план-факт таблицу, burn rate, income gap)
3. Добавить компактный итог в шапку Budget (план/факт за месяц одной строкой)
4. Убрать алерт кассового разрыва из Budget
