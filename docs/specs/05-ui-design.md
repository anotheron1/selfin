# UI/UX Дизайн: Личный платёжный календарь

*Документ формируется поэтапно в процессе проектирования интерфейса.*

## 1. Общая концепция и навигация (Layout & Navigation)
- **Визуальный стиль:** Свежий, современный, вызывающий "вау-эффект" дизайн. Используется связка **React + Tailwind CSS + Shadcn UI**. Интерфейс минималистичный, чистый, с поддержкой Темной темы (Dark Mode). Цветовая палитра сдержанная, акценты на важных финансовых показателях.
- **Навигация (Mobile-first подход):** Отказ от громоздкого сайдбара в пользу **Нижней панели навигации (Bottom Navigation Bar)**. Это обеспечит идеальную эргономику на смартфонах и лаконичный вид на ПК.
  - Панель состоит ровно из 4-х иконок: 📊 *Дашборд*, 📅 *Бюджет*, 💰 *Фонды*, ⚙️ *Настройки*.
- **Быстрый ввод (Киллер-фича):** На любом экране в правом нижнем углу всегда доступна плавающая кнопка "Плюс" (Floating Action Button, FAB). При нажатии открывается компактное модальное окно для ввода (с фокусом на поле ввода). Поддерживается шорткат (`Ctrl + N`). Ввел сумму и категорию -> нажал `Enter` -> окно закрылось, данные мгновенно улетели на сервер.

## 2. Главный экран (Дашборд / Аналитика)
Этот экран спроектирован для мгновенного считывания состояния финансов "в моменте". Основной фокус на тактике, а не долгосрочной стратегии.

- **Hero-метрика (Главная цифра):** Крупным жирным шрифтом выводится `Текущий баланс: X руб.`.

- **Двойной зарплатный горизонт** (под главной цифрой, три строки):

  | Строка | Пример | Назначение |
  |---|---|---|
  | До зп (28 мар): **4 200 ₽** | низшая точка | хватит ли до зарплаты |
  | После зп (28 мар): **54 200 ₽** | стартовый капитал | с чем войду в следующий период |
  | До зп (15 апр): **6 800 ₽** | низшая точка | выдержу ли второй период |

  Если в горизонте только одна зп — отображаются первые две строки.
  Если зп-событий нет — показывается `Прогноз конец месяца: Y руб.` (fallback).
  Строка «после зп» отображается с пониженной яркостью (dimmed) — вспомогательный контекст.

- **Алерт "Кассовый разрыв":** Красная карточка `⚠️ Внимание, [Дата] ожидается кассовый разрыв: -[Сумма] ₽` появляется если в горизонте (до второй зп или до конца месяца) баланс уходит в минус. Горизонт намеренно расширен до второй зп, чтобы разрыв между зарплатами не оставался невидимым.

- **Блок "План-Факт" (Прогресс-бары):** Линейные горизонтальные полоски-индикаторы по основным категориям расходов. Отображают: `Название категории | [||||||    ] 60% | Потрачено X из Y`. Цвета прогресса (зеленый → желтый → красный) меняются по мере приближения к лимиту плана. Стратегические виджеты (вроде "дефицита цели дохода") исключены для сохранения фокуса на тактическом контроле.

## 3. Экран "Календарь и Бюджет" (Ядро)
Выполнен в конфигурации "Понедельная Таблица-Список" (Weekly Ledger) для идеального считывания прогресса по расходам.
- Экран разделен на горизонтальные блоки-сворачиваемые списки: "Неделя 1", "Неделя 2", "Итоги месяца".
- В каждом блоке построчно отображаются запланированные и фактические траты (Слева название и Категория, Справа — микро-прогресс или `Факт / План`).
- **Сценарий корректировки факта:** Кликнув на любую строчку в расписании (например, "Бензин 3000"), пользователь открывает нижнее всплывающее окно (Bottom Sheet на мобильном / Modal на десктопе), где:
  1. Корректирует сумму по факту ("не 3000, а 2800").
  2. Нажимает "Сохранить" -> Строка тут же обновляется, остаток средств пересчитывается без перезагрузки экрана.

## 4. Экран "Целевые фонды" (Копилки)
- **"Кармашек" (Буфер):** В самом верху экрана крупно и красиво выводится сводная цифра свободных, нераспределенных средств: `В Кармашке: 50 000 ₽`. Это главный источник для пополнения фондов.
- **Интерфейс Копилок:** Реализован в виде сетки стильных квадратных **Карточек**.
- **Содержимое Карточки:**
  - Название цели (например, "Отпуск").
  - Круговой или линейный прогресс-бар: `Собрано 40к из 100к (40%)`.
  - *Умная статистика*: Вывод плановой даты закрытия цели, рассчитанной исходя из текущей скорости пополнения.
  - *Задел на масштабирование*: В будущем в карточке появятся настройки процентного отчисления (например, "автоматически забирать 20% из Кармашка в эту цель").

## 5. Экран "Настройки / Справочники"
Максимально плоский и функциональный раздел из-за отсутствия мульти-счетов.
- **Управление Категориями:** Простой список (Доходы/Расходы). Возле каждой категории находится ползунок-тумблер `Обязательная трата`.
- **Управление Снимками (Snapshots):** Блок с историей слепков бюджетов. Кнопка `Сделать снимок плана на текущий месяц` для фиксации стартовой точки отсчета.
- **Интеграции (Future-proof):** Блок для включения/выключения парсера ИИ и привязки Telegram-бота.
