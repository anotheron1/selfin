# Бизнес-требование: расчёт свободных денег

## Определение

**Свободные деньги** — сумма, которую пользователь может безопасно потратить на хотелки
и копилки, не поставив под угрозу обязательные платежи.

## Модель учёта

### Перевод в копилку = реальный расход

FUND_TRANSFER — это расход. Деньги уходят со счёта и больше не участвуют в расчёте
свободных средств. Балансы копилок НЕ вычитаются из баланса счёта отдельно.

### Формула баланса счёта (факт)

```
Баланс = checkpoint
       + Σ(factAmount всех INCOME)
       − Σ(factAmount всех EXPENSE)
       − Σ(factAmount всех FUND_TRANSFER)
```

Учитываются только исполненные события (FACT-записи или записи с factAmount != null).
Стартовая точка — последний BalanceCheckpoint; если чекпоинта нет — от нуля.

### Формула свободных денег (кармашек)

```
Кармашек (сейчас) = Баланс счёта − просроченные обязательные расходы
```

Просроченные обязательные = PLAN-записи с приоритетом HIGH, тип EXPENSE, статус PLANNED,
дата от начала месяца до вчера включительно. Это резерв: деньги, которые точно придётся
потратить, но ещё не потрачены.

### Горизонты прогноза

На странице «Копилки» показываются прогнозы на разных горизонтах:

**После обязательных расходов** — максимум, который теоретически можно пустить в копилки:
```
= Кармашек
+ Σ(plannedAmount будущих PLAN INCOME, статус PLANNED)
− Σ(plannedAmount будущих PLAN EXPENSE HIGH, статус PLANNED)
```

**После всех расходов** — минимум при полном выполнении плана:
```
= Кармашек
+ Σ(plannedAmount будущих PLAN INCOME, статус PLANNED)
− Σ(plannedAmount будущих PLAN EXPENSE, статус PLANNED)
− Σ(plannedAmount будущих PLAN FUND_TRANSFER, статус PLANNED)
```

### Кассовый разрыв

Кассовый разрыв = первый будущий день, когда нарастающий баланс (от текущего факта)
уходит в минус при последовательном применении ВСЕХ событий (обязательных и необязательных).

### Принцип: факт вытесняет план

Когда событие исполняется:
1. Создаётся FACT-запись с factAmount
2. PLAN-запись переходит в статус EXECUTED
3. PLAN(EXECUTED) **выпадает** из прогнозной части (не участвует в прогнозах)
4. FACT уже отражён в балансе счёта

Дельта между планом и фактом автоматически корректирует свободные деньги.

## Пример: месяц глазами пользователя

### Начальные данные (1 марта)

Checkpoint: 10,000 руб.

| Дата   | Событие    | Тип           | Приоритет | План    |
|--------|------------|---------------|-----------|---------|
| 5 мар  | Зарплата   | INCOME        | HIGH      | 100,000 |
| 10 мар | Аренда     | EXPENSE       | HIGH      | 30,000  |
| 15 мар | Коммуналка | EXPENSE       | HIGH      | 6,000   |
| 20 мар | Интернет   | EXPENSE       | MEDIUM    | 500     |

### 1 марта, утро. Ничего не исполнено.

```
Баланс счёта      = 10,000 (только checkpoint)
Кармашек           = 10,000 (нет просроченных обязательных)
После обязательных = 10,000 + 100,000 − 30,000 − 6,000 = 74,000
После всех         = 74,000 − 500 = 73,500
```

Пользователь видит: «У меня будет ~74,000 свободных на хотелки и копилки».

### 5 марта. Зарплата пришла: FACT 95,000 (−5,000 от плана).

```
Баланс счёта      = 10,000 + 95,000 = 105,000
Кармашек           = 105,000 (нет просроченных)
После обязательных = 105,000 − 30,000 − 6,000 = 69,000
После всех         = 69,000 − 500 = 68,500
```

Зарплата ниже плана → свободных на 5,000 меньше. Автоматическая корректировка.

### 10 марта. Аренда оплачена: FACT 28,000 (−2,000 от плана).

```
Баланс счёта      = 105,000 − 28,000 = 77,000
Кармашек           = 77,000
После обязательных = 77,000 − 6,000 = 71,000
После всех         = 71,000 − 500 = 70,500
```

Аренда дешевле → свободных на 2,000 больше.

### 10 марта, вечер. Перевод в копилку «Отпуск» 20,000.

```
Баланс счёта      = 77,000 − 20,000 = 57,000 (FUND_TRANSFER = расход)
Кармашек           = 57,000
После обязательных = 57,000 − 6,000 = 51,000
После всех         = 51,000 − 500 = 50,500
```

Копилка не вычитается отдельно — перевод уже уменьшил баланс.

### 15 марта. Коммуналка оплачена: FACT 7,000 (+1,000 от плана).

```
Баланс счёта      = 57,000 − 7,000 = 50,000
Кармашек           = 50,000
После обязательных = 50,000 (все обязательные оплачены)
После всех         = 50,000 − 500 = 49,500
```

Коммуналка дороже → свободных на 1,000 меньше.

### 12 марта (альтернативный сценарий). Коммуналка ещё не оплачена, но просрочена.

Если бы на 12 марта коммуналка (дата 15 мар, HIGH) ещё не была оплачена и дата уже прошла:

```
Баланс счёта      = 57,000
Просроченные      = 6,000 (коммуналка: HIGH, PLANNED, дата < сегодня)
Кармашек           = 57,000 − 6,000 = 51,000
```

## Правила расчёта по компонентам

### Dashboard: currentBalance

Текущий баланс = checkpoint + нарастающий итог FACT-записей.
Прошлые PLAN(EXECUTED) **не учитываются** — их вклад уже в FACT-записях.
Прошлые PLAN(PLANNED) **не учитываются** — деньги не потрачены, в балансе не убываем.

### Dashboard: балансовые прогнозы (до зп / после зп)

Стартуем от currentBalance. Прибавляем только PLAN(PLANNED) события в горизонте.
PLAN(EXECUTED) и FACT-записи не участвуют — они уже в currentBalance.

### Dashboard: кассовый разрыв

По-дневная итерация от currentBalance. Для каждого события:
- FACT-записи: берём factAmount
- PLAN(PLANNED): берём plannedAmount
- PLAN(EXECUTED): **пропускаем** (уже учтены через FACT)

### Analytics: кассовый календарь

Начальный баланс = checkpoint + «мостик» от checkpoint до начала месяца.
Мостик считается по тем же правилам: FACT → factAmount, PLAN(PLANNED) → plannedAmount,
PLAN(EXECUTED) → пропускаем.

### Funds: кармашек

Variant B: FUND_TRANSFER вычитается из баланса. Копилки отдельно не вычитаются.
